# Shared Components

## Purpose
This folder contains reusable components that do not belong to a single feature area.

## Analogy
Think of this folder as common equipment: useful pieces live here when several rooms in the application need them.

## Contents
- `LocationSelect.jsx` renders location selection UI.
- `PageHeader.jsx` renders a reusable page heading pattern.

## How It Fits in KERNO
Shared components reduce duplication across profiles, dashboards, catalog, and request workflows.

## Maintenance Notes
Keep this folder selective. If a component is used by only one feature, keep it in that feature's component folder.
