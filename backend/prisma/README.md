# Prisma

## Purpose
This folder contains the database schema, seed script, and migration history for the PostgreSQL database used by KERNO.

## Analogy
Think of this folder as the database blueprint room: it defines the shape of the data and how that shape evolves.

## Contents
- `schema.prisma` defines the Prisma data model, relationships, enums, and client generator.
- `seed-demo.js` populates a small, professional dataset for live demos and presentations (`npm run db:seed` / `db:seed:demo`).
- `seed-massive.js` populates a large development dataset used to exercise the core marketplace flow at scale (`npm run db:seed:massive`).
- `migrations/` contains versioned database migrations generated by Prisma.
- `migration_lock.toml` records the migration provider used by Prisma.
- `INSTALL_POSTGRES_PRISMA.md` documents local PostgreSQL and Prisma setup.

## How It Fits in KERNO
The backend service layer uses Prisma to read and write users, supplier profiles, store profiles, categories, products, and contact requests.

## Maintenance Notes
Update `schema.prisma` before generating migrations. Do not edit individual migration folders casually after they have been shared.
