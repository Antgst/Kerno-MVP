# API Docs

## Purpose
This folder documents the REST API exposed by the KERNO backend.

## Analogy
Think of this folder as the API contract binder: it explains how clients can interact with the backend.

## Contents
- `API_SUMMARY.md` summarizes the current backend API.
- `BACKEND_API_S2.md` preserves an earlier backend API reference for historical comparison.

## How It Fits in KERNO
The frontend service layer depends on the backend API contract documented here and in the runtime Swagger output.

## Maintenance Notes
Keep current API documentation aligned with backend routes and module Swagger files. Mark historical references clearly when they are not the active contract.
