# Sprints

## Purpose

This folder contains Stage 4 sprint planning, sprint reviews, and retrospective documentation for KERNO.

## Contents

- `SPRINT_PLAN_STAGE4.md` documents the sprint organization, priorities, responsibilities, and MVP scope.
- `SPRINT_REVIEWS_RETROSPECTIVES.md` records delivered work, challenges, and improvement notes for each sprint.

## How It Fits in KERNO

These documents support the Stage 4 requirement to prove that the MVP implementation was planned, tracked, reviewed, and adjusted through short development iterations.

## Maintenance Notes

Keep sprint notes aligned with GitHub issues, pull requests, the GitHub Project board, and the final review evidence.
