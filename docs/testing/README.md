# Testing Docs

## Purpose
This folder documents the testing strategy, regression coverage, and supporting test evidence for KERNO.

## Analogy
Think of this folder as the quality logbook: it records how the product is checked and what the latest test passes covered.

## Contents
- `FULL_REGRESSION_TEST_PLAN.md` describes the regression test plan.
- `FULL_REGRESSION_TEST_REPORT.md` records regression test results.
- `TESTING_EVIDENCE.md` stores testing evidence notes.
- `STAGE4_TEST_EVIDENCE.md` records the final Stage 4 technical and functional validation evidence.
- `postman/` contains Postman and Newman resources.
- `test_postman_S2/` contains earlier Postman artifacts and screenshots.

## How It Fits in KERNO
Testing documentation complements backend pytest coverage and frontend Playwright checks by explaining what quality signals are expected.

## Maintenance Notes
Keep generated artifacts separate from stable testing instructions. Update plans and reports when test scope or product behavior changes.
