# Engineering Notes

## Purpose
This folder contains internal engineering notes for the KERNO codebase.

## Analogy
Think of this folder as the engineering notebook: it captures observations that help maintain quality over time.

## Contents
- `TECHNICAL_REVIEW_NOTES.md` contains technical observations and follow-up notes.
- `STAGE4_MANUAL_REVIEW_CHECKLIST.md` prepares the final Stage 4 technical manual review.

## How It Fits in KERNO
Engineering notes complement architecture, API, database, security, and testing documentation by capturing broader implementation observations.

## Maintenance Notes
Keep notes factual, actionable, and tied to current repository behavior. Move durable architecture decisions into the architecture docs when they become settled.
