# Audits

This folder contains targeted audit notes created during project stabilization and review preparation.

Think of this folder as a focused quality notebook: it records what was checked, what was changed, what was intentionally left unchanged, and which validations were used.

## Documents

- `quality-audit-2026-07-03.md` documents the frontend quality audit performed before manual review preparation.
- `visual-audit-2026-07-03.md` documents the frontend visual consistency audit performed after manual review.
- `semgrep-audit-2026-07-03.md` documents the Semgrep security/code-pattern audit and links the CSRF finding to issue #232.
- `technical-audit-2026-07-07.md` documents a full-stack technical audit (architecture, code quality, security, performance, reliability, accessibility, dependencies, compliance) covering both backend and frontend.

## Maintenance

Add a new audit note here when a focused quality, security, visual, or technical review is performed and needs to be kept as repository evidence.
